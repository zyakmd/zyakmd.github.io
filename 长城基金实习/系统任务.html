# 遇到的问题

  • 数据读取，按位读取

  • 数据零散，建一个总表还是每个业务搞一张表？

    • 每个业务一张表，或者说每种数据类型一张表
    • 根据名字检测确定要输到哪张表，表名取文件名前三个，即是 OFD、OFI、等
    • 读取的时候配置从 xx 到 yy 的上述包含项，根据查询的时间获取。还可以做对象选择的判断（xx，yy 要符合，这个信息也是从文件名来，定义个 consumer 和 provider 属性）
    • 从数据库下载文件就根据 类型、时间、提供者、消费者 四个条件进行获取。

  • 把 OFD_22.ini 文件能设定对应的格式，读取

    • 62=N,16,2, 家庭资产，Penates
    • 上为例，首个是字段序号 = 类型、长度、小数长度、定义名称、字段名

  • 中文字符流占位问题。

    • 用的是 GB2312 编码，中文占两个字节，格式中描述为长度 2，类比于 .size ()，但实际指的是字节数。
    • 解决 1：单个字符做判断，属于中文编码的则长度 ++
    • 解决 2：本质是编码的问题，将字节流读取成 String 的时候（反序列化。序列化是对象到字节流），将其逻辑抽象为了长度一（因为底层用的 char，一个 char 算一个长度，但其实一个 char 占两个字节）。根本上就是将 String 序列化为 byte 数组后截取，再转回来。

  • 小数时，要特别做数据转换（保证了前端的精度）

    • BigDecimal 不用科学计数法，要用 toPlainString () 而不是 toString ()

  • 插入数据不报错，不抛异常，但失败，往往时某字段处理出问题了

  • sql 查询结果在使用 Map 集合接收时，集合中 map 值为 null 的数据 key 值不展示

    • https://segmentfault.com/a/1190000022179860

  • 有很多跳过式的错误，出了问题，但不抛异常也不报错

  • SQL 语句拼接庞大，解锁传输上限？要设置批次。

# 数据设计

设计一个文件信息预览界面，使用 Bootstrap 框架，美观！多用圆框淡色的交互图标

进入后先发送一个获取数据的请求，得到一个 List<Object> 数据（应该会经过 SpringBoot 序列化为 json 格式？我的表述都是后端 java 的形式，请自行处理为前端对应的格式）

根据 list 中的部分信息（文件名、日期、时间，创建人，接收人）。默认 20 条信息一页，可在页面更改，这些数据可以排序。这些数据的展示位于界面的中间，在上方还有个搜索选项，搜索会发送一个请求（携带搜索框中输入的文本），获取一个新的 List<Object> 数据，获取后刷新并并将列表替换为新的数据。

在这些数据的最右侧有个 “查看 / 修改” 按钮。点击后生成第一个弹窗并发送一个请求（包含上面数据的文件名），返回的是 List<List<String>> list 。这个弹窗用来展示这些返回数据，最上面是一些最开始数据的固定信息。主体部分是一个动态表格， list[0] 是其第一行，也是信息行，表示每列数据的含义 ，其余的每个 list 都是对应的一行数据数据。

因此根据 list[0].size() 在弹窗（占当前网页大小的 2/3 屏）生成一个动态的展示表格，表格有 list[0].size() 列， list.size() 行，每一列的宽度由 list[0].get(i) 动态决定，过长时使用滚动条。

在弹窗的上方，留有几行固定信息位，同时有个新增按钮。点击后又弹出第二个窗口，此窗口只有两列， list[0].size() 行，左侧显示 list[0].get(i) 的信息，右侧为空的文本栏。行数过多时有滚动栏，可以上下滑动。最下方有确认和取消按钮，点击取消则不做更改，确认按钮按绑定一个 post 方法，将右列的信息以 List<String> 的形式传给后端。取消或确认后都关闭第二个弹窗。同时刷新第一个弹窗，重新获取数据填充。

在第一个弹窗的每行最右侧还有一个修改按钮，点击后其展示界面和上面新增界面一致（即复用 ui），但会将当前行信息填到右列每行。同时下方的新增按钮换成修改按钮，点击后同样将右列作为 List<String> 传给后端。关闭并刷新第一个弹窗的逻辑同新增一样。
